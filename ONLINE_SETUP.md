# オンライン化の準備

## 決定した方式

- 家族共通のメールアドレスとパスワードでログイン
- ログイン後に「おかあ・りょういち・しゅんや」を選択
- 管理操作はおかあ用PINで保護
- 将来は子どもごとの個別ログインへ移行できる構造にする

## データ構成

- `families`: 家族
- `family_users`: ログインアカウントと家族の関連
- `family_members`: おかあ・子ども2人のプロフィール
- `tasks`: 宿題・勉強、お手伝い
- `task_completions`: 日付・子どもごとの完了記録
- `calendar_events`: 家族の予定
- `family_admin_settings`: おかあ用管理PINの設定

テーブル作成SQLは `supabase/schema.sql` にあります。家族以外からデータを読み書きできないよう、すべての共有テーブルでアクセス制御を有効にします。

## 次に必要なもの

1. Supabaseアカウント
2. 家族用の新規Supabaseプロジェクト
3. プロジェクトURL
4. 公開用の匿名キー

## Supabaseでの実行順

1. `supabase/schema.sql` を SQL Editor で実行して、データの表を作る
2. `supabase/online-upgrade.sql` を SQL Editor で一度だけ実行して、家族用ログインから安全にデータを作れるようにする
3. アプリで家族共通のメールアドレスとパスワードを登録する

プロジェクトURLと公開用キーはブラウザアプリへ設定できます。データベース管理用の秘密鍵はブラウザへ入れません。

## 移行方針

1. 現在の「バックアップを保存」でローカルデータを保存
2. オンライン接続後、初回だけバックアップを読み込む
3. データベースへ予定・タスク・チェック記録を移す
4. PC・スマホ・タブレットで同じ内容が見えることを確認
